### 一、项目整体逻辑（基于文件夹结构+Tauri2.0内网通讯软件通用架构推导）

该项目是基于 **Tauri2.0** 开发的跨平台内网通讯软件，核心架构分为两层：

1. **前端层（Web）**：基于 TypeScript + Vite 构建，负责 UI 交互（设备列表、消息收发界面）、前端状态管理、用户操作逻辑，通过 Tauri 的 IPC 机制与 Rust 层通信；

2. **Rust 层（src-tauri/）**：负责系统级能力落地，包括内网网络通信（UDP/TCP）、跨平台兼容、本地数据存储、文件操作、端口管理等核心能力；

3. **工程化支撑**：通过 prettier/rustfmt 做代码格式化、pre-commit 做提交校验、Cargo/npm 做依赖管理，保障代码规范和构建流程。

### 三、未实现的功能点（基于内网通讯软件完整能力模型+文件夹结构线索缺失推导）

以下功能无明确实现线索，属于未落地范畴（需结合 `IMPLEMENTATION_PLAN.md`/阶段报告进一步精准验证）：

#### 1. 高级通信能力（核心功能缺失）

- 内网群聊/多节点组通信（仅支持点对点，无组播/群聊协议）；

- 大文件分片传输+断点续传（仅支持小文件，无分片、失败重传逻辑）；

- 内网音视频通话（无音视频编解码、实时流传输、降噪/回声消除逻辑）；

- 跨网段内网通信（仅支持同网段，无中继节点/内网穿透机制）；

- 端到端加密（E2EE）+ 数字签名（仅基础对称加密，无高级安全防护）；

- 离线消息暂存/上线拉取（无离线状态下的消息缓存机制）。

#### 2. 交互体验优化（前端体验缺失）

- 富文本消息（仅纯文本，无图片/表情包/格式化文本支持）；

- 消息记录同步/备份（仅本地存储，无导出/导入/多设备同步）；

- 全局快捷键/系统托盘常驻（无后台运行、全局消息提醒热键）；

- 多语言适配（i18n）+ 主题定制（深色/浅色模式、自定义皮肤）；

- 消息已读/未读标识、撤回/编辑功能。

#### 3. 系统级增强能力（Rust 层能力缺失）

- 开机自启（跨平台开机自启配置未落地）；

- 完善的日志系统（无日志采集、分级、导出、异常定位）；

- 性能监控（无通信延迟、带宽/CPU/内存占用监控面板）；

- 错误重试/降级机制（通信失败无自动重试、弱网适配策略）；

- 进程守护（无主进程崩溃自动重启机制）。

#### 4. 扩展与运维能力（生态/管理能力缺失）

- 插件系统（无法扩展自定义功能插件）；

- 开放 API（无 HTTP/RPC 接口，无法对接其他内网工具）；

- 设备权限管理（无黑白名单、访问控制、设备备注/分组）；

- 通信审计（无消息审计日志、操作行为记录）；

- 远程指令执行（无内网设备远程命令下发、执行结果回传）；

- 自动更新（无 Tauri 自动更新机制，需手动下载安装包升级）。
